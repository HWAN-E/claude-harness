---
name: 세션 마무리 자동 인계 작업
description: 사용자가 종료 신호 발화하면 별도 요청 없이 다음 세션 인계 작업을 자동 수행
type: feedback
---
사용자가 세션 종료 신호를 보내면 (예: "내일 할게" / "내일 봐" / "오늘 마무리" / "수고했어" / "이만 끝" / "자야겠다" / "그만하자" / "내일 이어서" 등) **별도 요청 받지 않고 즉시** 아래 인계 작업을 자율 수행한다.

**Why:** 사용자가 매번 "다음 세션 이어갈 수 있게 처리해줘" 라고 요청하기 번거로움. 종료 신호만으로 자동 처리되기를 원함 (2026-04-29).

**How to apply (체크리스트, 위에서 아래로)**:

1. **HANDOFF 파일 작성** (모든 프로젝트 공통, 최우선)
   - 경로: `<project-root>/.claude/HANDOFF/<YYYY-MM-DD>_<HHMM>-<slug>.md`
   - 양식 (기존 archive 파일 참고):
     - `# 세션 인계 — <한 줄 제목>`
     - `## 작업 목적과 맥락` — 이번 세션이 왜 시작됐고 무엇을 풀려고 했나
     - `## 완료된 작업` — 산출물/변경 목록 (파일 경로 + 줄수 포함)
     - `## 다음 세션이 즉시 해야 할 액션` — "없음" 또는 구체 명령/파일경로
     - `## 핵심 사실` — 다음 세션이 잊지 말아야 할 것 (자주 쓰는 ID, 경로, 규칙)
     - `## 미해결 / 주의사항` — 알면서 미뤄둔 것, 함정
     - `## 관련 파일 경로` — 표 형태로 정리
   - 다음 세션 시작 시 SessionStart hook 이 자동 로드하므로 사용자가 별도 지시 안 해도 됨

2. **미커밋 변경 점검 + commit + push** (변경 있는 경우만)
   - `git status --short` 로 변경 파일 확인
   - 변경 유형별로 짧은 commit message 작성 (`feat/fix/docs/chore` prefix)
   - `git push origin <branch>` (현재 브랜치 자동 감지)
   - `.env`/secret 파일 staging 안 되게 주의
   - **사용자 명시 컨펌 후에만 진행** (글로벌 work_style 규칙 우선)

3. **PROJECT_STATUS.md 갱신** (있는 프로젝트만)
   - "진행 이력" 섹션에 오늘 날짜 항목으로 한 일 요약 (1~3 bullet)
   - 막힌 곳/시행착오/현재 위치 명시
   - "다음 세션 시작점" 박기 (구체 명령/URL/파일경로 수준)

4. **메모리 갱신** (필요한 경우만)
   - 진행 중인 큰 작업이 있으면 `project_<주제>_progress.md` 만들거나 갱신
   - 종결된 작업의 progress 메모는 삭제 또는 결정 메모로 흡수
   - MEMORY.md 인덱스 동기화

5. **todo list 정리**
   - 완료된 항목 completed 마킹
   - 다음 세션에 이어갈 항목만 남기기 (또는 비우고 HANDOFF/PROJECT_STATUS 로 일임)

6. **요약 보고** — 표 형태로 짧게:
   - 살아있는 리소스/서비스 (있는 경우, 비용 발생 여부 명시)
   - HANDOFF 파일 경로
   - 다음 세션 시작점 1~2줄
   - 짧은 푯말로 마무리 ("푹 쉬어", "내일 봐" 등)

**스킵 조건**:
- 변경 사항이 전혀 없으면 commit/push 단계는 생략 (빈 commit 만들지 X)
- PROJECT_STATUS.md 가 없는 프로젝트면 그 단계 생략
- 사용자가 명시적으로 "그냥 종료해" / "정리 안 해도 돼" 라고 하면 모두 생략 (단 HANDOFF는 가능하면 작성)

**주의**:
- push 는 외부 영향이지만 인계 목적상 필요. 단 force push, 다른 브랜치 push, hooks 우회 (`--no-verify`) 는 절대 자동 X. **work_style.md 의 사전 컨펌 규칙 우선.**
- HANDOFF 파일명은 `YYYY-MM-DD_HHMM-<짧은-slug>.md` 패턴 권장 (mtime 정렬 안정)
