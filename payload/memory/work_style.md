## 자율성

기본은 **알아서 진행**한다 — 단계마다 확인받지 않는다.
다만 아래 카테고리에 해당하면 정지하고 사전 확인을 받는다.

## 사전 확인이 필요한 작업 (되돌리기 어려운 것)

- 파일·디렉토리 **삭제**
- `git commit` / `git push` / `git rebase` / `git reset --hard` / force push
- DB 마이그레이션, 테이블 drop, 대량 데이터 변경 SQL
- 외부 시스템 영향: 외부 API 호출, 메시지/이메일 발송, 결제, 배포
- 의존성 큰 리팩터링 (여러 파일에 걸친 이름 변경 등)

## 사전 확인 양식 (한 줄)

```
[확인] 작업: <무엇> / 영향: <어디까지> / 롤백: <어떻게> — 진행할까요?
```

## 롤백 경로 보장

모든 변경은 롤백 가능해야 한다.

- **파일 변경** — 작업 시작 전, 변경 대상에 커밋되지 않은 변경이 있으면 사용자에게 commit 또는 stash 권유
- **파일 삭제** — 가능하면 trash 폴더로 이동(예: `.harness-trash/<timestamp>/`), 즉시 `rm`/`Remove-Item -Force` 금지
- **설정 변경** — 백업 후 변경 (`*.bak.<timestamp>`)
- **DB 변경** — 변경 전 dump 또는 마이그레이션 down 스크립트 확보

**Why:** 자율성은 가져가되, 되돌릴 수 없는 분기에서만 멈춘다.
**How to apply:** 위 카테고리에 들어가면 무조건 정지 → 한 줄 컨펌 양식 → 사용자 응답 후 진행.
